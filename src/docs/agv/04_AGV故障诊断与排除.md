# AGV 故障诊断与排除手册

## 概述

本文档提供 AGV 常见故障的诊断方法和排除步骤，帮助运维人员快速定位和解决问题。

## 1. 故障分类

| 故障等级 | 定义 | 处理时限 |
|----------|------|----------|
| 一级（紧急） | 影响安全或停产 | 立即处理 |
| 二级（严重） | 功能丧失 | 4 小时内 |
| 三级（一般） | 性能下降 | 24 小时内 |
| 四级（轻微） | 轻微异常 | 计划处理 |

## 2. 动力系统故障

### 2.1 电机过热

**现象：**
- 电机温度 > 80°C
- 速度明显下降
- 保护性停机

**诊断步骤：**
```
1. 检查负载是否超限
2. 检查电机散热片清洁度
3. 检查环境温度是否过高
4. 检测电机三相电流平衡性
5. 测量电机绕组绝缘电阻
```

**处理方法：**
- 停机冷却 15-30 分钟
- 清理电机散热片
- 降低运行环境温度
- 必要时更换电机

### 2.2 驱动轮打滑

**现象：**
- 行驶轨迹偏移
- 里程计读数异常
- 定位精度下降

**诊断步骤：**
```
1. 检查地面是否有油污
2. 检查轮胎磨损程度
3. 检查轮胎气压（充气式）
4. 检查制动器是否抱死
```

**处理方法：**
- 清理地面油污
- 更换磨损轮胎
- 调整轮胎气压
- 检修制动器

### 2.3 电池故障

**电池无法充电**
```
诊断：充电器输出正常，但电池电压不上升
可能原因：
□ 电池过放保护触发
□ 电池连接线松动
□ 电池内部故障
□ 充电接口接触不良

排除步骤：
1. 检查充电线连接
2. 测量电池端电压
3. 尝试激活电池（涓流充电）
4. 更换电池测试
```

**电池容量下降**
```
诊断：充电时间正常，但使用时间明显缩短
可能原因：
□ 电池老化
□ 充电不充分
□ 环境温度影响
□ 深度过放造成损伤

排除步骤：
1. 进行完整的充放电循环
2. 测量实际容量
3. 检查单体电池电压一致性
4. 更换老化电池组
```

## 3. 导航系统故障

### 3.1 激光传感器故障

**故障现象：**
- 定位丢失
- 路径偏移告警
- 障碍物检测失效

**诊断方法：**
```
1. 检查传感器镜头是否有污物
2. 检查传感器安装角度
3. 检查反射板是否移位
4. 测量传感器供电电压
5. 检查通讯信号波形
```

**处理措施：**
- 清洁传感器镜头
- 重新校准角度
- 调整反射板位置
- 更换传感器

### 3.2 陀螺仪漂移

**故障现象：**
- 直行时方向偏移
- 原地旋转角度不准
- 长时间运行累计误差增大

**校准步骤：**
```
1. 将 AGV 置于平整地面
2. 在手动模式启动校准程序
3. 保持 AGV 静止 30 秒
4. 缓慢旋转 360°（约 60 秒）
5. 等待校准完成
6. 测试直行和转向精度
```

### 3.3 地图匹配失败

**故障现象：**
- 调度系统报"定位异常"
- AGV 原地徘徊
- 无法执行路径任务

**排查流程：**
```
1. 检查当前环境是否与地图一致
   □ 地面是否变化（油漆、胶带）
   □ 是否有新增固定障碍物
   □ 反射板是否被遮挡

2. 检查定位参数
   □ 定位置信度 > 0.7
   □ 传感器数据有效
   □ 里程计数据正常

3. 重新定位操作
   □ 将 AGV 移至已知位置点
   □ 执行强制定位
   □ 验证定位精度
```

## 4. 通讯系统故障

### 4.1 网络连接中断

**故障现象：**
- 调度系统显示"离线"
- 任务无法下发
- 状态数据无法上报

**诊断步骤：**
```
1. 检查 AGV 网络指示灯
2. 测量网络线连接
3. 检查交换机端口
4. 测试 AGV  IP 配置
5. 检查防火墙设置
```

**快速恢复：**
```bash
# 重启 AGV 网络服务
1. 断开网络连接 5 秒
2. 重新连接
3. 等待 30 秒自动重连
4. 如未恢复，重启 AGV 控制系统
```

### 4.2 通讯延迟过高

**诊断标准：**
- 平均延迟 > 100ms：异常
- 丢包率 > 1%：异常

**排查方向：**
- 网络带宽是否拥塞
- 是否有干扰信号
- 天线安装位置是否合理
- AP 覆盖是否存在盲区

## 5. 传感器系统故障

### 5.1 障碍物检测失效

**危险：可能导致碰撞！**

**诊断步骤：**
```
1. 检查激光传感器清洁度
2. 检查传感器视角是否被遮挡
3. 验证检测区域参数设置
4. 进行传感器自检
5. 检查输出信号
```

**安全确认方法：**
```python
# 手动测试障碍检测
1. 设置安全区域（人员远离）
2. 在 AGV 行驶路径放置障碍物
3. 观察 AGV 是否减速/停止
4. 测试各方向检测灵敏度
```

### 5.2 碰撞传感器误报

**故障现象：**
- 行驶中频繁急停
- 特定区域必报故障
- 无障碍物时报警

**排查方向：**
- 地面反光干扰
- 金属物体反射干扰
- 传感器阈值设置过低
- 传感器老化导致灵敏度漂移

## 6. 控制系统故障

### 6.1 控制系统死机

**应急处理：**
```
1. 按下急停按钮
2. 等待 10 秒
3. 关闭主电源
4. 等待 30 秒
5. 重新上电启动
6. 如频繁死机，记录日志联系厂家
```

### 6.2 固件异常

**诊断依据：**
- 自检报错 E0xx 系列
- 功能异常但无明确故障点
- 升级后出现新问题

**处理方法：**
- 备份当前配置
- 回滚至上一版本固件
- 或升级至最新稳定版
- 重置为出厂设置

## 7. 故障诊断工具

### 7.1 诊断接口

通过调试端口连接：
```bash
# 查看实时状态
$ agv_monitor --status

# 查看故障日志
$ agv_monitor --fault-log

# 查看传感器数据
$ agv_monitor --sensor-data
```

### 7.2 自检程序

启动自检：
```
控制面板 → 系统设置 → 自检程序 → 执行
```

自检项目：
- 电机功能测试
- 传感器响应测试
- 通讯功能测试
- 安全回路测试

## 8. 疑难问题升级

以下情况需要联系厂家技术支持：

```
□ 控制系统硬件故障
□ 导航算法异常
□ 电机驱动板故障
□ 安全回路失效
□ 反复出现的同类型故障
```

报告时应提供：
- 设备型号和序列号
- 故障代码和完整日志
- 故障发生时的操作和环境
- 已尝试的排查步骤
