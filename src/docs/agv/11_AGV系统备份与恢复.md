# AGV 系统备份与恢复指南

## 概述

本文档规定 AGV 系统的数据备份策略和灾难恢复流程，确保系统数据安全。

## 1. 备份类型

### 1.1 备份分类

| 备份类型 | 频率 | 保留数量 | 说明 |
|----------|------|----------|------|
| 自动每日备份 | 每日凌晨 | 7 份 | 系统自动执行 |
| 手动完整备份 | 按需 | 无限制 | 重大变更前必须 |
| 配置快照 | 每次配置变更 | 10 份 | 自动生成 |
| 日志归档 | 每周 | 12 份 | 按月归档 |

### 1.2 备份内容

```
备份内容清单：
================================
1. 地图数据
   - 栅格地图文件
   - 拓扑地图配置
   - 站点数据库
   - 路径配置

2. 系统配置
   - AGV 参数配置
   - 调度参数
   - 安全参数
   - 网络配置

3. 用户数据
   - 用户账户
   - 权限配置
   - 操作日志

4. 运行数据
   - 任务历史
   - 告警记录
   - 统计报表

5. 软件环境
   - 应用程序
   - 固件版本
   - 依赖库
================================
```

## 2. 备份策略

### 2.1 备份时间安排

| 备份类型 | 时间 | 执行方式 |
|----------|------|----------|
| 每日增量备份 | 02:00 | 自动 |
| 每周完整备份 | 周日 02:00 | 自动 |
| 月度归档备份 | 每月最后一日 03:00 | 自动 |
| 变更前备份 | 变更前手动执行 | 手动 |
| 升级前备份 | 升级前 1 小时 | 手动 |

### 2.2 备份存储策略

```
存储层级：
================================
本地存储（AGV 本机）
  - 最新 3 份备份
  - 用于快速恢复

服务器存储（本地服务器）
  - 最新 30 份备份
  - 用于服务器故障恢复

云端存储（可选）
  - 最新 12 份月度备份
  - 用于灾难恢复

离线存储（物理备份）
  - 季度备份复制到硬盘
  - 异地保存
================================
```

### 2.3 备份验证

每次备份完成后必须验证：

```
□ 备份文件完整性检查
□ 备份文件大小对比
□ 抽样数据正确性验证
□ 恢复测试（每月至少 1 次）
```

## 3. 备份操作流程

### 3.1 手动完整备份

```bash
# 登录调度系统管理后台
# 选择"系统管理" -> "备份管理"
# 点击"创建完整备份"
# 填写备份描述
# 选择存储位置
# 点击"开始备份"
# 等待完成并验证
```

### 3.2 地图数据备份

```bash
# 备份指定 AGV 的地图数据
$ agv_backup --map --agv AGV-01 --output /backup/maps/

# 备份所有地图
$ agv_backup --map --all --output /backup/maps/

# 备份输出示例：
# [2026-04-17 10:00:00] Starting map backup...
# [2026-04-17 10:00:05] AGV-01 map backup complete
# [2026-04-17 10:00:10] AGV-02 map backup complete
# [2026-04-17 10:00:15] All maps backup complete
# Total size: 125MB
```

### 3.3 配置备份

```bash
# 导出系统配置
$ agv_config --export --output /backup/config/

# 导出内容包括：
# - system_config.yaml
# - network_config.yaml
# - safety_params.yaml
# - agv_params.yaml
```

### 3.4 数据库备份

```bash
# 备份 PostgreSQL 数据库
$ pg_dump -h localhost -U agv_user -d agv_ops \
  --format=c --compress=9 \
  --file=/backup/db/agv_ops_20260417.dump

# 验证备份
$ pg_restore --list /backup/db/agv_ops_20260417.dump
```

## 4. 恢复操作流程

### 4.1 系统恢复类型

| 恢复类型 | 适用场景 | 恢复时间 |
|----------|----------|----------|
| 配置恢复 | 配置错误 | 5-10 分钟 |
| 单机恢复 | 单台 AGV 故障 | 15-30 分钟 |
| 地图恢复 | 地图损坏 | 10-20 分钟 |
| 全量恢复 | 服务器故障 | 1-2 小时 |
| 灾难恢复 | 重大故障 | 4-8 小时 |

### 4.2 配置恢复

```bash
# 从备份文件恢复配置
$ agv_config --restore \
  --backup /backup/config/config_20260415.yaml \
  --target system

# 重启服务使配置生效
$ systemctl restart agv-scheduler
```

### 4.3 地图恢复

```bash
# 恢复指定 AGV 的地图
$ agv_restore --map --agv AGV-01 \
  --backup /backup/maps/AGV-01_map_20260410.bak

# 验证地图完整性
$ agv_map --verify --agv AGV-01
```

### 4.4 全量恢复

```
恢复步骤：
================================
1. 准备工作
   - 确认新服务器/存储已就绪
   - 准备系统安装介质
   - 准备所有备份文件

2. 系统安装
   - 安装操作系统
   - 安装数据库
   - 安装调度系统
   - 配置网络

3. 数据恢复
   - 恢复数据库
   - 恢复地图数据
   - 恢复配置数据
   - 恢复用户数据

4. 验证测试
   - 启动服务
   - 检查所有 AGV 连接
   - 执行测试任务
   - 验证数据完整性

5. 恢复运行
   - 切换到生产模式
   - 监控运行状态
   - 记录恢复完成时间
================================
```

### 4.5 恢复验证清单

```
恢复完成后必须验证：
================================
□ 数据库连接正常
□ 所有 AGV 在线
□ 地图数据完整
□ 调度功能正常
□ 任务下发正常
□ 告警功能正常
□ 历史数据可查询
□ 备份系统正常运行
================================
```

## 5. 灾难恢复计划

### 5.1 灾难场景

| 灾难类型 | 影响范围 | RTO | RPO |
|----------|----------|-----|-----|
| 单台 AGV 故障 | 单台设备 | 1 小时 | 0 |
| 服务器硬盘损坏 | 全部 AGV | 4 小时 | 1 天 |
| 服务器完全损坏 | 全部 AGV | 8 小时 | 1 天 |
| 数据中心级灾难 | 全部系统 | 24 小时 | 7 天 |

> RTO: 恢复时间目标
> RPO: 恢复点目标

### 5.2 灾难恢复流程

```
触发条件：服务器无法在 4 小时内恢复
================================
1. 宣布灾难（应急总指挥）
   - 评估灾难程度
   - 决定启动灾难恢复
   - 通知相关人员

2. 资源调配
   - 调配恢复用服务器
   - 准备备份介质
   - 安排恢复人员

3. 执行恢复
   - 按全量恢复流程执行
   - 优先恢复调度核心功能
   - 逐步恢复各 AGV

4. 业务恢复
   - 验证基础功能
   - 小规模试运行
   - 逐步恢复生产

5. 事后总结
   - 分析灾难原因
   - 改进备份策略
   - 更新恢复流程
================================
```

## 6. 备份管理

### 6.1 备份存储位置

| 存储位置 | 容量 | 用途 |
|----------|------|------|
| /backup/local/ | 100GB | 本地备份 |
| /backup/server/ | 500GB | 服务器备份 |
| /backup/cloud/ | 1TB | 云端备份 |
| /backup/offline/ | - | 离线归档 |

### 6.2 备份清理策略

```
清理规则：
- 每日备份：保留 7 天
- 每周备份：保留 30 天
- 每月备份：保留 365 天
- 季度备份：永久保留
- 手动备份：由管理员决定

清理时间：每月 1 日执行
清理前必须确认备份完整性
```
